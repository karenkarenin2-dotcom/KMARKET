# KMARKET — проектный контекст для Claude

**KMARKET by KareninTeam** — аналитика рынка World of Warcraft. Первый и пока
единственный модуль — **Жетон WoW (EU)**: копим историю цены, считаем аналитику
и отвечаем на один главный вопрос — **брать сейчас или ждать**.

Автор — Карен, профессиональный аниматор, начинающий программист; весь код
пишет Claude. Язык общения — русский. Соседние проекты студии:
`../KARENINTOOLS/` (KTOOLS, панель для Maya) и `../KARENIN_SITE/` (лендинг) —
оттуда унаследованы модель именования и визуальный язык, НЕ выдумывать свои.

## ГЛАВНЫЙ ФАКТ, из которого следует вся архитектура

**Blizzard API отдаёт ТОЛЬКО текущую цену жетона.** Эндпоинт
`/data/wow/token/index?namespace=dynamic-eu` возвращает две величины: `price`
(в меди) и `last_updated_timestamp` (в миллисекундах). Истории у Blizzard нет
вообще — ни за день, ни за час.

Отсюда: **сборщик важнее дашборда.** Дашборд можно переписать за вечер,
пропущенные часы истории не восстанавливаются ничем. Любое изменение,
затрагивающее сбор, оценивать по критерию «не может ли это молча остановить
сбор».

**Такт обновления — ровно 20 минут** (измерено 2026-07-22 на живых данных:
EU 11:02:44 → 11:22:44, US 11:01:32 → 11:21:32). У каждого региона своё
смещение внутри такта. Наш опрос раз в 10 минут — это ровно две попытки на
каждую точку: одна лишняя нужна, потому что крон Actions опаздывает.

## Решения, зафиксированные Кареном (сессия 2026-07-22)

| Решение | Выбор | Почему |
|---|---|---|
| Имя | KMARKET | растёт в полноценную рыночную аналитику |
| Сбор данных | GitHub Actions, крон `*/10` | работает, когда ПК выключен; ноль затрат |
| Репозиторий | **публичный** | на приватном Actions даёт 2000 мин/мес, а ~4400 запусков в месяц упрутся в лимит и сбор молча встанет; на публичном минуты безлимитные |
| Дашборд | FastAPI + своя вёрстка | фирменный стиль важнее скорости старта; Streamlit отвергнут |
| Регионы | EU (основной) + US | US собираем как ГИПОТЕЗУ об опережающем индикаторе |
| Сценарий Карена | **покупает жетон за золото** | «хорошо» = цена НИЗКАЯ; сигналы затачиваются под поиск дна, не под продажу |
| Горизонт | запасается на месяцы вперёд | нет дедлайна → перцентили на 90–365 дней, сигналы редкие и сильные |
| Аналитика | статистика + индикаторы | ML-прогноз отложен: он требует месяцев истории и всё равно ошибается |
| Время | данные в UTC; график в `Asia/Omsk`, сезонность в `Europe/Paris` | Карен в Омске (UTC+6), EU-сервера на CEST (UTC+2) — разница 4 часа; без разделения тепловая карта врёт на треть суток |
| Алерты | Telegram (`@kmarketwowbot`) + дашборд | пуш работает из Actions даже при выключенном ПК |
| Персонализация | НЕ делаем | дашборд про рынок, не про кошелёк |
| Архив истории | загрузить разово из внешнего источника | перцентили без месяцев истории бессмысленны |
| Статус | «сначала для себя, но строим как продукт» | продуктовые именование/код/дизайн, без лендинга и донатов пока |

## Как работать с этим проектом

- **ДО написания кода выпытывать у Карена ключевые решения короткими вопросами
  с вариантами** (процесс из KTOOLS, одобрен: «вопросы дешевле переделок»).
- Давать правки **цельными, готовыми версиями** — Карен смотрит результат глазами.
- Не изобретать визуальный язык — он уже есть (см. ниже и `../KARENIN_SITE/CLAUDE.md`).
- Секреты — только в `.env` (локально) и GitHub Secrets (в CI). В коде их нет.
  История: до 2026-07-22 ключи Blizzard лежали в `config.py` открытым текстом —
  файл удалён, секрет перевыпущен.

## Архитектура

Три слоя, независимых по времени жизни:

```
kmarket/
  config.py     секреты из окружения/.env, пути, пояса, список регионов
  blizzard.py   OAuth + /data/wow/token/index          ← НОЛЬ зависимостей
  storage.py    CSV-история, дедуп по времени Blizzard  ← НОЛЬ зависимостей
  collect.py    точка входа сборщика                    ← НОЛЬ зависимостей
  analytics/    перцентили, сезонность, вердикт, бэктест (pandas)
  web/          FastAPI + шаблоны дашборда
data/
  history/<регион>/<ГГГГ-ММ>.csv   наши замеры (коммитит Actions)
  archive/<регион>/*.csv           разовый внешний бутстрап
```

- **Сборщик обязан работать на голой стандартной библиотеке.** `pip install`
  в Actions — это лишняя минута на каждый из ~4400 месячных запусков и лишняя
  точка отказа из-за чужого релиза. Зависимости (pandas, FastAPI) нужны только
  локальному дашборду. Это правило не нарушать даже ради удобства.
- **Опрашиваем каждые 10 минут, хотя цена меняется ~раз в 20.** Крон в Actions
  не гарантирует точность: в пиковые часы запуск задерживается или пропускается.
  Лишние опросы бесплатны (дубли отсеиваются по `updated_utc`), пропуск — нет.
- **CSV, а не база.** Читается глазами, дифается в git (репозиторий и есть
  хранилище), год замеров ≈ 52 тыс. строк — для pandas ничто. База потребовала
  бы хостинга, то есть денег и ещё одной точки отказа.
- **Дедупликация по `updated_utc`** — времени самой Blizzard, не времени опроса.
- Месячный файл всегда переписывается целиком отсортированным (через `.tmp` +
  атомарный `replace`): на 4-5 тыс. строк мгновенно, а git видит diff в одну строку.
- **Ненулевой код возврата у сборщика** при отказе API — обязателен: молча
  вставший сбор не восстановить.

## Правила игры

Мы читаем официальный Blizzard Game Data API, который Blizzard сама даёт
разработчикам. Игрового клиента не касаемся и не автоматизируем — нарушением
было бы именно это. Никаких аддонов, макросов и вмешательства в игру.
